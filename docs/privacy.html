Privacy Policy

Last updated: June 2026

1. Information We Collect

Our application collects only the minimal data required to support secure user authentication, event registration management, and dynamic content rendering. This data is transmitted to and stored on our remote backend infrastructure.

Authentication Data

  • Email address and password (for standard email-based registrations)
  • Sign in with Apple Data: If you authenticate using Sign in with Apple, we collect your unique stable user identity token, full name, and your email address (or the anonymized Private Relay email alias provided by Apple).
  • Full name (collected strictly during user sign-up)
  • Supabase user ID and system metadata assigned by the backend infrastructure

Event Registration Data

  • Student name and email address
  • Optional custom registration fields (depending on event requirements): Phone number, Registration number, Department, Year, and additional notes

Uploaded Media & App Content

  • Images selected by users or administrators for event covers, announcement posters, or board member headshots
  • Application text data including announcements, event details, board member profiles, and "About Us" configurations

What We Do Not Collect: The application does not access, collect, or request permissions for your precise location, personal contacts, health data, advertising identifiers, background photos, or third-party telemetry/analytics libraries.

2. How We Use the Information

Collected information is used exclusively to deliver the core functional experiences of the application:

  • Authentication: Email and password credentials, or Sign in with Apple security identifiers, are processed via Supabase Auth to establish secure user accounts, manage active sessions, grant registration capabilities, and provision administrative access levels.
  • Event Management: Registration information is compiled into internal rosters to track event participation and manage event workflows.
  • Media & Display: Uploaded images and structured app content are fetched from our database to render announcement feeds, event cover cards, active roster details, and board member profile layouts within the user interface.

3. Whether the Data is Shared

We value user privacy and strictly limit data exposure to our explicit functional architecture:

  • Primary Infrastructure: Data is shared exclusively with our internal Supabase backend services, including Supabase Auth, dedicated relational database tables (announcements, events, registrations, board_members, app_config), and the Supabase Storage bucket adg-assets.
  • Public Media Availability: Images uploaded to the adg-assets storage bucket are served via public URLs. Anyone possessing the direct asset URL can access the corresponding image file.
  • No Commercial Sharing: Personal registration tracking and user profiles are utilized strictly internal to the ecosystem. We do not sell, rent, trade, or distribute data to third-party advertising networks, analytics trackers, or data brokers.

4. Data Security and Cloud Storage

Your data transmission and cloud persistence are secured utilizing standard engineering protocols:

  • Transit Security: All API and data connections to our remote Supabase backend are encrypted over HTTPS using the Supabase Swift client SDK, authenticating via system projectURL and anonKey parameters.
  • Session Security: Authenticative session persistence is managed securely via the Supabase SDK, emitting local tokens to maintain standard login states. Image downloading is handled via standard URLSession handling utilizing native device caching protocols.
  • Storage Architecture: No local database frameworks (such as Core Data or CloudKit) are deployed. Data relies completely on the transit security and storage protection rules configured on our remote cloud host.

5. User Rights and Data Deletion

We respect your rights over your data records. The application provides native controls for signing up, signing in, and signing out of your active session.

In full compliance with App Store Guideline 5.1.1, the application features an automated, self-service account deletion trigger directly within the user account/profile interface. Initiating this process instantly and permanently deletes your authentication account from Supabase Auth, along with all associated historical records, personal metadata, and event registration entries across our database infrastructure.

Alternatively, users retain full rights to request manual purges or modifications to their specific data records by issuing a request through our official communication channels listed below.
